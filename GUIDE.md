# ACT Console 使用指南

> 写给所有 ACT 模板的用户。从安装到跑通，一篇看完。

---

## 这个插件是什么

ACT Console 是 ACT 笔记系统的仪表盘。打开 Obsidian 后，它在侧边栏给你一个统一的操作界面——不需要在文件夹之间来回翻，就能看到今天该做什么、知识库长什么样、目标推进到哪了。

它围绕 ACT 的三层结构工作：

| 层级 | 对应文件夹 | 插件里做什么 |
|------|----------|------------|
| **Action 行动** | `10-Action/` | 看任务、记进展、切状态 |
| **Card 知识** | `20-Card/` | 看知识库全貌、索引引用、最近新增 |
| **Time 方向** | `30-Time/` | 看十二周目标、愿景、周记历史 |

---

## 第一步：安装插件

将 `act-console` 插件文件夹（包含 `main.js`、`manifest.json`、`styles.css`）复制到你 Vault 的 `.obsidian/plugins/act-console/` 目录下。

重启 Obsidian，在设置 → 第三方插件中启用 **ACT Console**。

启用后，左侧边栏会出现插件图标，点击即可打开仪表盘。

---

## 第二步：配置文件夹路径

> 打开 Obsidian 设置 → 第三方插件 → ACT Console

**如果你使用的是标准 ACT 模板，需要调整以下路径：**

ACT 模板的文件夹结构和插件默认值有几处不同。下面列出需要手动修改的地方（所有路径设置都支持下拉选择，输入时会弹出匹配列表）：

### 行动 Tab

| 设置项 | 插件默认值 | ACT 模板实际路径 | 是否需要改 |
|-------|-----------|----------------|-----------|
| 收件箱 | `+` | `+` | 不用改 |
| 聚焦承诺 | `10-Action/11-Focus-聚焦承诺` | `10-Action/11-Focus-聚焦承诺` | 不用改 |
| 活跃跟进 | `10-Action/12-Active-活跃跟进` | `10-Action/12-Active-活跃跟进` | 不用改 |
| 将来也许 | `10-Action/13-Maybe-将来也许` | `10-Action/13-Maybe-将来也许` | 不用改 |
| 随想文件夹 | `+/ACT随想` | 模板中没有此文件夹 | 可新建或留空 |

### 时间 Tab

| 设置项 | 插件默认值 | ACT 模板实际路径 | 是否需要改 |
|-------|-----------|----------------|-----------|
| 日志 | `30-Time/34-Daily-日志` | `30-Time/34-Daily-日志` | 不用改 |
| 周记 | `30-Time/33-Weekly-每周` | `30-Time/33-Weekly-每周` | 不用改 |
| 十二周 | `30-Time/32-12Week-十二周` | `30-Time/32-12Week-十二周` | 不用改 |
| 愿景 | `30-Time/31-Vision-愿景` | `30-Time/31-Vision-愿景` | 不用改 |

### 知识 Tab

**这里有差异，需要手动修改：**

| 设置项 | 插件默认值 | ACT 模板实际路径 | 需要改成 |
|-------|-----------|----------------|---------|
| 卡片根目录 | `20-Card` | `20-Card` | 不用改 |
| 索引卡 | `20-Card/22-IndexCard-索引卡` | `20-Card/21-IndexCard-索引卡` | **改成 21** |
| 阅读卡 | `20-Card/23-BibCard-阅读卡` | `20-Card/22-BibCard-阅读卡` | **改成 22** |
| 核心卡 | `20-Card/24-MainCard-核心卡` | `20-Card/23-MainCard-核心卡` | **改成 23** |
| 新卡暂存 | `20-Card/21-NewCard-新卡暂存` | 模板中没有此文件夹 | 可新建或留空 |

> ACT 模板的 Card 编号是 21-索引、22-阅读、23-核心，没有新卡暂存文件夹。插件默认值是 22-索引、23-阅读、24-核心、21-新卡暂存。**必须改，否则知识面板数据会对不上。**

## 第三步：让任务被插件识别

这是最关键的一步。**插件通过 `a-任务笔记` 标签来识别哪些笔记是任务。**

你的任务笔记的 frontmatter 中需要包含这个标签：

```yaml
---
tags:
  - a-任务笔记
created: 2026-03-28
deadline: 2026-04-15
priority: 2
AI 备注: 简短描述任务核心
---
```

### 插件识别的任务笔记结构

插件会读取以下二级标题区块的内容：

| 区块标题 | 插件用途 | 可用的别名 |
|---------|---------|-----------|
| `## 下步行动` | 在面板上显示行动清单 | `## 下一步行动`、`## 行动清单` |
| `## 进展记录` | 在面板上显示最近进展，也是"记录进展"的写入位置 | — |
| `## 背景目标` | 任务的背景信息 | — |

一个完整的任务笔记示例：

```markdown
---
tags:
  - a-任务笔记
created: 2026-03-28
deadline: 2026-04-15
priority: 2
AI 备注: 完成课程前三章逐字稿
---

## 下步行动

- [ ] !!! 完成第一章初稿
- [ ] !! 准备配图素材
- [ ] ! 和设计师确认排版

## 进展记录

### 2026-03-28 14:30
今天完成了大纲，明天开始写正文。

## 背景目标

课程的前三章是基础介绍部分，需要在 4 月中旬前完成。
```

**行动优先级标记：**
- `!!!` = 最优先（面板中高亮显示）
- `!!` = 重要
- `!` = 一般

### 如果你已有任务笔记

ACT 模板中的示例任务笔记使用 `## 行动清单` 而非 `## 下步行动`——这没问题，插件同时支持这两种标题。

但你需要确保 `tags` 中包含 `a-任务笔记`。如果你之前用的是 `a1-聚焦承诺` 之类的标签，**需要额外加上 `a-任务笔记`**，两者可以共存：

```yaml
tags:
  - a-任务笔记
  - a1-聚焦承诺
```

## 第四步：让十二周目标被识别

插件会自动计算当前处于哪个十二周周期，然后去 `32-12Week-十二周/` 文件夹中寻找对应的目标文件。

**文件命名规则：** `YYYY-Yx.md`

| 周数范围 | 对应文件 |
|---------|---------|
| W01-W12 | `2026-Y1.md` |
| W13-W24 | `2026-Y2.md` |
| W25-W36 | `2026-Y3.md` |
| W37-W52 | `2026-Y4.md` |

> ACT 模板中的示例文件叫 `26Y1-12周目标-示例.md`，这个命名不会被插件自动识别。你可以保留它作为参考，但实际的目标文件需要按 `2026-Y1.md` 格式命名。

**目标文件中的格式：**

```markdown
### G1 - 完成课程前三章

这个周期要拿到的结果：前三章逐字稿完成，配图到位。

相关任务：
- [[2601-课程制作]]
- [[课程内容选题]]

### G2 - 建立每周发布节奏

每周至少发布 1 篇内容。

- [[内容排期]]
```

**关键规则：**
- 用 `### G1 - 标题` 格式定义目标（支持 G1、G2、G3...）
- 目标区块内的 `[[双链]]` 会自动关联行动层的任务笔记（前提是那个笔记有 `a-任务笔记` 标签）
- 关联成功后，面板上会在每个目标下面显示任务的执行状态

## 第五步：创建知识层数据视图（可选）

知识面板的"知识层总览"区域有四个卡片类型的计数（核心卡、阅读卡、索引卡、新卡暂存），点击数字可以跳转到对应的 Base 数据视图。

如果你想启用这个功能，需要在 `20-Card/` 下创建一个 `.base` 文件。

在 `40-storage/44-base-数据视图/` 中已有一些 `.base` 示例，你可以参考创建，或者直接在 `20-Card/` 下新建 `知识总览.base`：

```yaml
views:
  - type: table
    name: 核心卡总览
    filters:
      and:
        - file.inFolder("20-Card/23-MainCard-核心卡")
        - file.ext == "md"
    order:
      - file.name
      - created
  - type: table
    name: 阅读卡总览
    filters:
      and:
        - file.inFolder("20-Card/22-BibCard-阅读卡")
        - file.ext == "md"
    order:
      - file.name
      - created
  - type: table
    name: 索引卡总览
    filters:
      and:
        - file.inFolder("20-Card/21-IndexCard-索引卡")
        - file.ext == "md"
    order:
      - file.name
  - type: table
    name: 新卡暂存
    filters:
      and:
        - file.inFolder("20-Card/21-NewCard-新卡暂存")
        - file.ext == "md"
    order:
      - file.name
```

然后在插件设置 → 知识 Tab 中，将数据视图路径设置为：
- 核心卡：`20-Card/知识总览.base#核心卡总览`
- 阅读卡：`20-Card/知识总览.base#阅读卡总览`
- 索引卡：`20-Card/知识总览.base#索引卡总览`

> 注意：这里的文件夹路径要和你实际的文件夹名一致。上面的示例用的是 ACT 模板的编号（21/22/23），不是插件默认值。

## 四个面板详解

配置完成后，打开插件看看每个面板在做什么。

### 🎯 Today · 聚焦

**每天打开的第一个页面。**

| 区域 | 数据来源 | 说明 |
|------|---------|------|
| 任务清单 · 今日 | `10-Action` 三个文件夹 | 读取带 `a-任务笔记` 标签的笔记中用 `!` 标记的行动项 |
| 固定日程 | 滴答清单 API | 今天的日程（需启用滴答清单集成） |
| 今日已完成 | 周记 + 行动笔记 | 汇总今天在各处勾选完成的事项 |
| 最近随想 | `+/ACT随想/` | 随想笔记列表，可快速跳转到今日日志 |

任务清单区域有三个子标签页：**聚焦承诺 / 活跃跟进 / 将来也许**，对应三个文件夹。右上角的 `+` 按钮可以快速新建任务笔记。

### ⚡ Action · 行动

**任务的深度管理台。**

左侧是所有任务笔记列表，右侧是选中任务的详情：

- **执行状态**：推进中 / 等待中 / 暂停搁置 / 已完成。一键切换，自动写入 frontmatter tags
- **下步行动**：读取 `## 下步行动`（或 `## 行动清单`）区块的内容
- **记录进展**：直接在面板里写进展，保存后插入到 `## 进展记录` 区块顶部。支持五种标签——进展、判断、卡点、情绪、下一步
- **最近进展**：显示最近 3 条进展记录

> **核心概念**：文件夹管承诺程度（聚焦 / 活跃 / 也许），标签管执行状态（推进中 / 等待中 / 暂停 / 完成）。两者正交。

### 💎 Card · 知识

**知识库的全景视图。**

- **主题索引**：展示索引卡文件夹下的所有索引卡，按子文件夹（Topic-主题索引 / Person-人物索引）分组。每张索引卡旁显示被核心卡引用的次数
- **知识层总览**：各类卡片的数量统计，点击跳转 Base 数据视图
- **最近新增**：最近修改或创建的卡片

### 🧭 Time · 方向

**从长期视角审视方向。**

- **十二周目标**：读取当前周期目标文件，显示进度条和关联任务
- **周度流水**：本周工作的完成情况
- **愿景**：愿景文件夹中的文件列表
- **周记历史**：最近的周记列表

## 滴答清单集成（可选）

如果你使用滴答清单管理日程：

1. 打开插件设置 → 滴答清单 Tab
2. 开启「启用滴答清单」
3. 获取 Token：前往**网页版**滴答清单 → 点击「头像」→「设置」→「账户与安全」→「API 口令」→ 创建并复制
4. 粘贴到 Access Token 输入框

启用后，Today 页面会显示今天的滴答清单日程，完成的任务也会出现在「今日已完成」中。

---

## AI 技能（Skill）配置（可选）

如果你使用 Claude Code / Codex 等 AI CLI 工具：

1. 打开插件设置 → Skill Tab
2. **终端模式**：选择点击技能按钮时的行为
   - Termy 插件（Obsidian 内终端，推荐）
   - Terminal 插件（Obsidian 内终端）
   - 系统终端（弹出外部终端窗口，macOS / Windows 均支持）
   - 仅复制命令（复制到剪贴板，手动粘贴）
3. **CLI 工具**：选择 Claude Code 或 Codex，也可自定义命令模板
4. 在技能列表中添加你常用的 Skill 名称

## 常见问题

### 面板上看不到我的任务？

1. 确认笔记在 `10-Action` 的三个子文件夹之一
2. 确认 frontmatter 的 `tags` 包含 `a-任务笔记`（不是 `#a-任务笔记`，frontmatter 里不需要 `#`）
3. 确认文件是 `.md` 格式

### 知识面板数据不对？

最常见的原因是路径没改。ACT 模板的 Card 编号（21/22/23）和插件默认值（22/23/24）不同，需要在设置中手动调整。

### 十二周目标页面显示"当前周期目标尚未创建"？

检查文件命名。插件寻找的是 `2026-Y1.md` 格式，不是 `26Y1-目标名.md`。

### 点击知识总览的数字没反应？

需要先创建 `.base` 数据视图文件并在设置中配置路径（见第五步）。

### 怎么新建任务？

两个入口：
- Today 页面 → 任务清单右上角的 `+` 按钮
- Action 页面 → 任务列表上方的 `+` 按钮

新建的笔记会自动包含 `a-任务笔记` 标签和标准模板。

### 进展记录写到哪了？

在 Action 面板中写的进展会以 `### YYYY-MM-DD HH:MM` 格式插入到任务笔记的 `## 进展记录` 区块最顶部。如果笔记中没有这个区块，会自动创建。

## 给模板发布者的备忘

如果你要将 ACT Console 插件随模板一起分发，需要确保：

1. **插件文件**：将 `main.js`、`manifest.json`、`styles.css` 放入 `.obsidian/plugins/act-console/`
2. **预设配置**：在 `.obsidian/plugins/act-console/data.json` 中预设正确的文件夹路径，让用户开箱即用，不需要手动改路径：

```json
{
  "folders": {
    "inbox": "+",
    "focusAction": "10-Action/11-Focus-聚焦承诺",
    "activeAction": "10-Action/12-Active-活跃跟进",
    "maybeAction": "10-Action/13-Maybe-将来也许",
    "daily": "30-Time/34-Daily-日志",
    "weekly": "30-Time/33-Weekly-每周",
    "cycle": "30-Time/32-12Week-十二周",
    "vision": "30-Time/31-Vision-愿景",
    "card": "20-Card",
    "newCard": "",
    "indexCard": "20-Card/21-IndexCard-索引卡",
    "bibCard": "20-Card/22-BibCard-阅读卡",
    "mainCard": "20-Card/23-MainCard-核心卡",
    "thought": "+/ACT随想",
    "thoughtFile": "+/ACT随想/每日随想.md"
  }
}
```

3. **community-plugins.json**：确保 `act-console` 在启用列表中
4. **示例任务笔记**：确保示例任务笔记的 tags 中包含 `a-任务笔记`
5. **十二周目标示例**：建议同时提供一个 `YYYY-Yx.md` 格式的文件，让用户能直接在面板上看到效果
